 Customer Shopping Behavior Analytics
Project Overview

This project demonstrates an end-to-end data analytics workflow — from raw data processing to business intelligence reporting.

The objective is to simulate a corporate analytics environment where data is cleaned, structured, analyzed, and transformed into actionable insights for decision-making.

Workflow

1️⃣ Data Preparation & Modeling (Python)

Cleaned and standardized raw dataset

Handled missing values

Applied feature engineering

Created new analytical columns (age_group, purchase_frequency_days)

Prepared data for database storage

2️⃣ Data Analysis (MySQL)

Stored processed dataset in relational database

Wrote SQL queries for:

Revenue analysis

Customer segmentation

Product performance

Purchase behavior patterns

3️⃣ Visualization (Power BI)

Built interactive dashboard

Designed KPIs:

Total Customers

Total Revenue

Average Rating

Revenue by Category

Seasonal Trends

Enabled filtering and drill-down capabilities

4️⃣ Reporting & Insights

Identified high-value customer segments

Analyzed loyalty and purchase frequency

Extracted product performance trends

Delivered business recommendations

Tech Stack

Python (Pandas, NumPy)

MySQL

Power BI

DAX

Key Business Insights

Certain categories contribute significantly higher revenue

Subscription customers show stronger loyalty

Review ratings influence purchase patterns

Seasonal variations impact sales performance
